5. 11. 2025

main entry point: lending_market.move
    also a registry (a little bit)

    - user methods deposit, withdraw, requestWithdraw (the underlying tokens are staked)
    - putting C-tokens in serve as colalteral, you first need to obtain them
    - forgiving debt: socialized loss for bad debt
    - claimReward: 2 versions: iether take it for itself or redeposit it
    - borrow: P is lending market type (phantom), T is the type your borrowing, collateral is everything you have in bank

struct LendingMarket: some stuff is deprecated, prepared for future use, unused

Obligation:
    - user specific accounts..
    - list of all deposits, each deposit has backlink (index)

Reserve:
    - arrayIndex = where in the array it is
    - coin_type = typename. Like generic, but they dont want to have it generic for some reasons (it would propagate everywhere)

ReserveConfig:
    - ltv, open x close (close = liquidation)
    - weight: if you depositing suspicious tokens, you pay more
    - attributed..limit are not used
    - borrowing isolated asset => you're not allowed to borrow other stuff

loop obligations: we dont want to incetivise people to loop, i.e. deposit and borrow the same asset
    deposit USDc, borrow USDt, etc. they don't get reward when they loop
    this is done via the disabled_pairings_map
    but they might drop this in the future, maybe only keep for the exact same asset

Staking:
    - entry point: liquid_staking (also the registry)
    - lst = coin for staking (share)
    - there is no unstake period

LendingMarket
 - id, reserves, obligations, FeeReceivers

Reserve
 - id, marketID, coinType, price (in USD),
 - availableAmount, cTokenSUpply, borrowed_amount
 - totalSupply = available + borrowed - unclaimedFees

Obligation
 - marketID
 - list of deposits - at most one per coinType, all with nonzero amount, these are cTokens
 - list of borrows: the same..
 - deposited_value_USD = total value of all deposits
 - allowed_borrow_value_usd the value that allows to open the position, ltv <= ltv_open
 - unhealthy_borrow_value_usd the value that allows the position to exists without liquidation, ltv <= ltv_close
